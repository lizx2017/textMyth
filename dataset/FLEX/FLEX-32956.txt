Summary:
Flex 3.5 Menu - Submenu not working properly with more data
Description:
Flex 3.5 Menu - Submenu not working properly with more data. The menu items are displaced when the data in xml is more than 10 items. Attached the sample source code to replicate the bug.
Steps to reproduce:
1. Run the attached mxml file in Flex Builder and run the application by clicking the menu button.
2. The sub-items are not displayed appropriately as expected.
Actual Results:
> The sub-menu displayed is displaced and not rendered as expected.
Expected Results:
> The sub-menu should be displayed appropriately
Workaround (if any):
> Set Application height to fixed height (800px) this works but number of other things in application breaks (application resizing etc)
