Summary:
Text in textinput does not move while scrolling
Description:
How to reproduce:
Just select the text input at the bottom of the view and activate the softkeyboard. The keyboard will push the text input upwards accordingly. Type anything inside the text input and deactivate the soft keyboard.
When the softkeyboard is activated and causing the scroller to scroll up, the text inside the TextInput component does not scroll up as intended. This bug can be reproduced in every time, even with a mobile project.
