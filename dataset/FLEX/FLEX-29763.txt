Summary:
Not able to click on the components in search result grid of Watson Express application after configuring with QTP
Description:
Steps to reproduce:
1. Open http://fb-linux1.corp.adobe.com:8080/racecar/RunTimeLoading.html?automationswfurl=WatsonExpressFlex.swf in a browser where QTP is configured with Flex
2. Login to the application (please contact Lakshmi gunna (lgunna@adobe.com) for user id & password)
3. Click on Search box at the top
4. Click on Add Project, click down arrow to select Project
5.Select Family = "ESG", Product = "ABTS", Version = "3.0 and click OK button
6. Click Run button in the below
7. In the Search result, click on any row to open Bug View
Actual Results:
1. Not able to click on the row
2. Not able to click on "Customize" button in the below
3. Not able to click in the filter box
Expected Results:
1. When you click on a row in the search result grid, Bug View must be opened.
2. When you click on Customize button, must pop up a window, which has fields to cutomize the search result grid
3. Should be able to enter the text in the filter box at the below.
Note: For more information about the Expected behavior, you can do the same query in http://fb-linux1.corp.adobe.com:8080/racecar/ (not configured with QTP)
