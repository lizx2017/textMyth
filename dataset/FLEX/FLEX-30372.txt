Summary:
Unable to capture the New Flex Objects if its overlapped in the existing Flex Data Grid component - Urgent
Description:
Steps to reproduce:
Unable to capture the Flex Objects in the below cases:
1. While an FlexLink Bar is selected, an new Flex Title Window will be displayed - Unable to capture the Flex Button.
2. When an column is selected in the Flex Data Grid, an new Flex Button will be displayed - Unable to capture the Flex Button.
3. An Vertical Scroll bar in the Flex Data Grid is not captured.
Actual Results:
When an Action is performed in the Existing Flex Objects, the focusing is not properly moved to the newly displayed Flex Objects like Flex Title Window or Flex Button.
While doing the Object Spy, its returning only the existing Flex Data Grid position
Even the Flex Objects are not recognised using QTP 10.0
Unable to capture the Flex Objects, if an New Flex Objects are overlapped in the existing Flex Data Grid.
Even in the Recording Mode, the New Flex Objects are not captured in the Object Repository.
Expected Results:
New Flex Objects that are overlapped in the existing Flex Data Grid should be captured in the Object Repository.
Workaround (if any):
Comments: If anyone has resolved the above issues, please inform me immediately. This is an very critical/unexpected issue faced in the Flex Pages.
