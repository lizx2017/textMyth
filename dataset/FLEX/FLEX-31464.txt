Summary:
The ODG doesn't get back to the same state as the original display, if a column "+" sign, is closed and then opened again
Description:
Steps to reproduce:
1. Run the bug file, click the button to run the query.
2. On the column header, click on the "-" handler, the columns collapse.
3. now click on the "+" handler on the column, the query is resubmitted and the new results come up, but it doesn't layout the same as the original query.
Actual Results: The ODG doesn't come back to the same original state after interaction.
Expected Results: for a given query the ODG should display it in the same way.
Workaround (if any): set column widths on your own.
